{{ define "main" }}
<div class="categories-page">
    <header class="page-header">
        <h1 class="page-title">所有分类</h1>
        <p class="page-description">本站点共有 {{ len .Data.Terms }} 个分类</p>
    </header>
    
    <div class="categories-cloud">
        {{ $categories := .Data.Terms }}
        {{ if $categories }}
            {{ $max := len (index $categories.ByCount 0).Pages }}
            {{ $min := len (index $categories.ByCount.Reverse 0).Pages }}
            
            {{ range $name, $taxonomy := $categories }}
                {{ $count := len $taxonomy.Pages }}
                <!-- 计算权重和字体大小，可以根据需要调整公式 -->
                {{ $weight := div (sub (math.Log $count) (math.Log $min)) (sub (math.Log $max) (math.Log $min)) }}
                {{ $fontSize := add 0.8 (mul $weight 2.5) }}
                
                <a href="{{ .Page.Permalink }}" 
                   class="category-cloud-item" 
                   style="font-size: {{ $fontSize }}em;"
                   title="{{ $count }} 篇文章">
                    {{ .Page.Title }} <sup>{{ $count }}</sup>
                </a>
            {{ end }}
        {{ else }}
            <p class="no-content">暂无分类</p>
        {{ end }}
    </div>
</div>
{{ end }}

                   class="tag-cloud-item" 
                   style="font-size: {{ $fontSize }}em;"
                   title="{{ $count }} 篇文章">
                    {{ $name }} <sup>{{ $count }}</sup>
                </a>

            <p class="no-content">暂无标签</p>

</div>

5. 验证和测试

完成以上步骤后：

  1. 运行 hugo server 启动本地服务器
  2. 访问 http://localhost:1313/categories/ 查看分类列表页
  3. 访问 http://localhost:1313/tags/ 查看标签云页面
  4. 检查导航菜单中的链接是否正确工作

注意事项

  1. 模板优先级：Hugo 会优先使用这些专用模板（layouts/categories/terms.html 和 layouts/tags/terms.html），而不是通用的 layouts/_default/terms.html。

  2. 分类法配置：确保你的配置文件中正确定义了分类法：

    1
2
3

    taxonomies:
  category: categories
  tag: tags

  3. 内容文件：如果你的 content 目录下有 categories.md 或 tags.md，它们可能会覆盖这些自动生成的页面。如果遇到问题，可以考虑删除或重命名这些内容文件。

  4. 样式调整：你可以根据自己网站的主题风格，调整 CSS 中的颜色、间距、字体大小等属性。
