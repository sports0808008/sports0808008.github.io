Kafka安装

任务实现

前提条件：

需要启动zookeeper，请参考zookeeper步骤检查是否已启动成功

1、在hadoop102上解压缩包

[root@hadoop102 ~]# cd /opt/software

[root@hadoop102 software]# tar -xzf kafka.tar.gz -C /opt/module/

2、在hadoop102配置系统环境变量/etc/profile

[root@hadoop102 ~]# vi /etc/profile

在文件末尾添加如下配置

export KAFKA_HOME=/opt/module/kafka

export PATH=$PATH:$KAFKA_HOME/bin

[root@hadoop102 ~]# source /etc/profile

3、配置Kafka

为Kafka创建数据目录

[root@hadoop102 ~]# source /etc/profile

[root@hadoop102 ~]# cd $KAFKA_HOME

[root@hadoop102 kafka]# mkdir kafka-logs

然后打开config目录下的server.properties文件，修改日志目录为刚刚创建的目录:

[root@hadoop102 kafka]# cd $KAFKA_HOME/config

[root@hadoop102 config]# vi server.properties

找到并修改如下配置的值，其中broker.id的值需要保证在整个集群中是唯一的

broker.id=1

log.dirs=/opt/module/kafka/kafka-logs

zookeeper.connect=hadoop102:2181,hadoop103:2181,hadoop104:2181

将Kafka和环境变量拷贝到hadoop103、hadoop104并修改broker.id的值

[root@hadoop102 kafka]# cd /opt/module/

[root@hadoop102 module]# scp -rq kafka hadoop103:/opt/module/

[root@hadoop102 module]# scp -rq kafka hadoop104:/opt/module/

[root@hadoop102 module]# scp -rq /etc/profile hadoop103:/etc/

[root@hadoop102 module]# scp -rq /etc/profile hadoop104:/etc/

在hadoop103修改kafka的配置文件server.properties，将broker.id的值修改为2

[root@hadoop103 ~]# source /etc/profile

[root@hadoop103 ~]# cd $KAFKA_HOME/config

[root@hadoop103 config]# vi server.properties

修改后broker.id的值如下

broker.id=2

在hadoop104修改kafka的配置文件server.properties，将broker.id的值修改为3

[root@hadoop104 ~]# source /etc/profile

[root@hadoop104 ~]# cd $KAFKA_HOME/config

[root@hadoop104 config]# vi server.properties

修改后broker.id的值如下

broker.id=3

分别在hadoop102、hadoop103、hadoop104启动Kafka的后台守护进程

[root@hadoop102 ~]# kafka-server-start.sh -daemon $KAFKA_HOME/config/server.properties

[root@hadoop103 ~]# kafka-server-start.sh -daemon $KAFKA_HOME/config/server.properties

[root@hadoop104 ~]# kafka-server-start.sh -daemon $KAFKA_HOME/config/server.properties

4、验证Kafka

1）创建topic，创建名为test的topic，分区数1，副本1

[root@hadoop102 ~]# kafka-topics.sh –create –zookeeper hadoop102:2181,hadoop103:2181,hadoop104:2181 –replication-factor 1 –partitions 1 –topic test

2）查看topic的状态

[root@hadoop102 ~]# kafka-topics.sh –describe –zookeeper hadoop102:2181,hadoop103:2181,hadoop104:2181 –topic test

3）在client客户端节点上打开一个终端，使用ssh命令连接到hadoop102，发送消息

[root@hadoop102 ~]# kafka-console-producer.sh –broker-list hadoop102:9092 –topic test

4）在client客户端节点上重新打开一个终端，使用ssh命令连接到hadoop102，消费消息

[root@hadoop102 ~]# kafka-console-consumer.sh –bootstrap-server hadoop102:9092 –from-beginning –topic test
