1.ES6的新特性

​ ES6，全名ECMAScript 2015(欧洲计算机协会制造商)，是JavaScript编程语言的一个重要更新版本，引入了许多新特性和改进，使其更加强大和表达力丰富。

1.1. 不一样的变量声明：const和let

ES6推荐使用let声明局部变量，相比之前的var（无论声明在何处，都会被视为声明在函数的最顶部）
let和var声明的区别：

let表示声明变量，而const表示声明常量，两者都为块级作用域；const 声明的变量都会被认为是常量，意思就是它的值被设置完成后就不能再修改了：

如果const的是一个对象，对象所包含的值是可以被修改的，就是对象所指向的地址没有变就行：

有几个点需要注意：

◆let 关键词声明的变量不具备变量提升（hoisting）特性

就是在声明之前的代码中可以访问变量就叫做变量提升，var可以变量提升，但是没有值。

◆let声明的变量 和 const 声明的常量 只在最靠近的一个块中（花括号内）有效

◆当使用常量 const 声明时，请使用大写变量，如：CAPITAL_CASING

◆const 在声明时必须被赋值(必须给常量初始化)

1.2. 模板字面量

模板字面量允许您在字符串内使用反引号和${}`来嵌入表达式。

1.3. 箭头函数（Arrow Functions）

ES6 中，箭头函数就是函数的一种简写形式，使用括号包裹参数，跟随一个 =>，紧接着是函数体；

箭头函数最直观的两个个特点：

◆不需要 function 关键字来创建函数

◆可以省略return关键字

​ 细节：当你的函数有且仅有一个参数的时候，是可以省略掉括号的。当你函数返回有且仅有一个表达式的时候可以省略{} 和 return；

箭头函数 特点

箭头函数有以下几个特点：

  1. 简洁的语法：箭头函数使用 (参数) => {函数体} 的形式进行定义，相比传统函数定义方式更加简洁明了。

  2. 隐式返回：如果箭头函数的函数体只有一条表达式，可以省略大括号和 return 关键字，直接返回该表达式的结果。这种隐式返回的语法形式使得函数的定义更加简洁。

  3. 词法作用域的 this 值：箭头函数的 this 值是词法上绑定的，它会捕获当前环境的 this 值，并在函数内部使用。这与传统函数中的 this 值会根据函数的调用方式来确定不同，使得箭头函数在处理 this 值时更加直观和方便。

  4. 没有自己的 arguments 对象：箭头函数没有自己的 arguments 对象，也无法使用 arguments 访问传递给它的参数。这是因为箭头函数是词法上绑定的，不具备动态变化的特性。

  5. 不适用于构造函数：箭头函数不能用作构造函数，不能使用 new 关键字和使用 prototype 属性。这是因为箭头函数没有自己的 this 值，无法正确地绑定实例对象。

总体而言，箭头函数具有简洁的语法、隐式返回、词法作用域的 this 值等特点，适用于简单的函数定义和保持上下文的场景。然而，也要注意箭头函数的限制，如不能用作构造函数和没有自己的 arguments 对象。

1.5. for…of 和 for…in

for…of 和for…in 用于遍历一个迭代器，如数组：

只要区别 for ..In

在 JavaScript 中，for...of 和 for...in 是两种不同的循环遍历方式，用于迭代对象中的元素或属性。它们的区别如下：

  1. 使用范围：

    • for...of 循环适用于迭代可迭代对象（iterable），如数组、字符串、Set、Map 等。它会遍历对象中的元素值。
    • for...in 循环适用于遍历对象的可枚举属性。它会遍历对象中的键（key）。

  2. 遍历顺序：

    • for...of 循环按照迭代对象的顺序，逐个遍历其中的元素值。
    • for...in 循环遍历对象的属性时，并不保证按照特定顺序进行。对象的属性遍历顺序可能不同于它们在对象中定义的顺序。

  3. 遍历内容：

    • for...of 循环遍历对象中的属性值。
    • for...in 循环遍历对象中的键（key）。

  4. 遍历继承属性：

    • for...of 循环不会遍历对象的原型链上的属性。
    • for...in 循环会遍历对象的原型链上的所有可枚举属性。

  5. 其他限制：

    • for...of 循环无法获取遍历过程中的当前索引（index），也无法直接获取当前元素所在的集合对象。
    • for...in 循环遍历时，会包括对象的所有可枚举属性，包括原型链上的属性。如果要排除继承的属性，需要使用 hasOwnProperty() 方法进行过滤。

2.异步请求

异步请求请求的是后端的数据，数据传输的方式在最开始的时候是xml格式，后面发展为了json

Ajax（Asynchronous JavaScript and XML）是一种用于在网页上进行异步数据交互的技术。它允许您通过不必刷新整个网页来获取或发送数据，而只更新页面的一部分。这使得用户体验更加流畅，因为用户可以与页面交互而不受到页面刷新的干扰。以下是有关Ajax的概念和理解

2.1. 异步性质

Ajax的核心特征是它是异步的。这意味着在执行Ajax请求时，页面不会被阻塞，用户可以继续与页面交互，而不必等待请求完成。

2.2. XMLHttpRequest对象

XMLHttpRequest是js提供的内最对象，该对象用来发送异步请求。

早期 Ajax请求通常使用XMLHttpRequest对象创建。这个对象允许您发送HTTP请求（GET、POST等）并处理响应。

既然使用这种回调函数层层嵌套（又称：回调地狱）的形式存在缺点，ES6想了办法治它，所以就有了Promise的出现了。

3.2. Promise的基本用法

（1）书写格式

​ Promise对象是全局对象，你也可以理解为一个类，创建Promise实例的时候，要有那个new关键字。参数是一个匿名函数，其中有两个参数：resolve和reject，两个函数均为方法。resolve方法用于处理异步操作成功后业务；reject方法用于操作异步操作失败后的业务。

3.3. Promise对象三种状态

Promise对象有三种状态：

pending：刚刚创建一个Promise实例的时候，表示初始状态；

fulfilled：resolve方法调用的时候，表示操作成功；

rejected：reject方法调用的时候，表示操作失败；

状态只能从 初始化 -> 成功 或者 初始化 -> 失败。

上面的注释，讲清楚了一个Promise实例的状态改变情况。

初始化实例后，对象的状态就变成了pending；当resolve方法被调用的时候，状态就变成了：成功fulfilled；当reject方法被调用的时候，状态就会有pending变成失败rejected。

5.async和await关键字的使用

​ async和await关键字是ES7提出的新规范之一，利用这两者可以让我们的异步操作变得更加简洁。

 `async`和`await`是ES2017（ES8）引入的JavaScript关键字，用于简化异步操作的编写和处理。它们一起使用，可以使异步代码看起来更像是同步代码的写法，提升了代码的可读性和可维护性。

下面是async和await关键字的使用方式：

  1. async关键字：在函数前面加上async关键字声明一个异步函数。异步函数会返回一个Promise对象，该Promise对象的解析值（fulfilled状态的结果）是函数的返回值。通过async关键字声明的函数内部可以使用await关键字来等待和处理异步操作。

  2. await关键字：在async函数内部，可以在一个异步操作前加上await关键字，使其等待异步操作完成，并返回异步操作的结果。await关键字只能在async函数内部使用。使用await关键字的语句将会暂停当前函数的执行，直到异步操作完成并返回结果，然后继续执行函数并返回结果。

使用async和await关键字能够让异步代码以更直观的方式编写，而无需使用回调函数或处理Promise链。它们使得代码更加清晰、易读，并且能够有效地处理异步操作带来的回调地狱问题。

​ 当使用async关键字修饰一个函数时，它将返回一个Promise对象，这个Promise对象的解析值将是函数的返回值。通过在async函数中使用await关键字，可以等待一个异步操作完成，并将结果赋值给一个变量。

简单来说，async函数和await关键字的使用可以将异步操作以同步的方式进行编写和处理，避免了回调函数嵌套和Promise链的复杂性。

5.2. async关键字

​ async函数的返回值为 promise 对象，promise 对象的结果由 async 函数执行的返回值决定

◆当返回非Promise的数据类型，PromiseState作为成功来处理

◆当返回Promise数据类型,PromiseState根据Promise返回结果决定。

作用：当你的函数需要返回Promise对象的时候使用 async 来修饰。

5.3. await关键字

​ await 右侧的表达式一般为 promise 对象, 但也可以是其它的值， 如果表达式是 promise 对象, await 返回的是 promise 成功的值； 如果表达式是其它值, 直接将此值作为 await 的返回值

5.4. 注意

◆await 必须写在 async 函数中, 但 async 函数中可以没有 await

◆如果 await 的 promise 失败了, 就会抛出异常, 需要通过 try…catch 捕获处理
