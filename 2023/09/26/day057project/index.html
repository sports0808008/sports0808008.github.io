6.3. Mybaits-plus的概念与使用

官网：http://mp.baomidou.com/

MyBatis-Plus (opens new window)（简称 MP）是一个 MyBatis (opens new window)的增强工具，在 MyBatis 的基础上只做增强不做改变，为简化开发、提高效率而生（单表操作的王者）。

​ 原始的mybatis需要写sql，现在有了mybatis-plus，基本上可以不写sql（在操作单表数据的时候就可以完全不写sql），但是如果是连表查询还是要写sql语句。

​ 注意：并不是mybatis不使用了，不是使用mybatis-plus取代了mybatis，而是在不改变mybaits的使用的基础上进行了增强。

​ 补充：以前有一个映射框架叫做hibernate，这个框架是全自动的框架，就是可以不写sql语句，但是牺牲了性能。

​ 愿景****：我们的愿景是成为 MyBatis 最好的搭档，就像 魂斗罗 中的 1P、2P，基友搭配，效率翻倍。

11. MyBatis-Plus 提供的一个用于分页查询的工具类

QueryWrapper 是 MyBatis-Plus 提供的一个用于封装查询条件的工具类。它可以帮助我们方便地构建查询条件并与数据库进行交互。

QueryWrapper 提供了一系列的方法来处理查询条件，以下是其中一些常用的方法：

  • eq：使用等于条件查询，例如 queryWrapper.eq("column", value)。
  • ne：使用不等于条件查询，例如 queryWrapper.ne("column", value)。
  • like：使用模糊查询条件，例如 queryWrapper.like("column", value)。
  • in：使用 in 条件查询，例如 queryWrapper.in("column", values)。
  • between：使用 between 条件查询，例如 queryWrapper.between("column", value1, value2)。
  • orderBy：指定排序字段，例如 queryWrapper.orderByAsc("column") 或 queryWrapper.orderByDesc("column")。
  • select：指定查询的字段，例如 queryWrapper.select("column1", "column2")。

你可以根据具体的需求使用这些方法来构建查询条件。在你提供的代码中，已经使用了 like、eq 等方法来构建查询条件。

请注意，QueryWrapper 还有其他许多方法可用于处理查询条件，你可以根据需要查看 MyBatis-Plus 的官方文档以了解更多详情。

Page 是 MyBatis-Plus 提供的一个用于分页查询的工具类。它提供了一系列用于处理分页查询的属性和方法，帮助我们方便地进行分页操作。

使用 Page 类进行分页查询的基本步骤如下：

  1. 创建 Page 对象。在创建 Page 对象时，需要传入两个参数：当前页码和每页记录数。例如，Page<Employee> pageInfo = new Page<>(currentPage, pageSize);，其中 currentPage 表示当前页码，pageSize 表示每页记录数。

  2. 调用分页查询方法。一般情况下，我们使用 MyBatis-Plus 提供的 page 方法进行分页查询。该方法接受两个参数：要查询的分页对象和查询条件对象。例如，employeeService.page(pageInfo, queryWrapper);，其中 employeeService 是服务类对象，queryWrapper 是包含查询条件的 QueryWrapper 对象。

  3. 处理分页查询结果。分页查询之后，分页对象 Page 中会包含查询结果的相关信息，如总记录数、总页数、当前页记录列表等。你可以通过以下方法获取这些信息：

    • getTotal()：获取总记录数。
    • getPages()：获取总页数。
    • getCurrent()：获取当前页码。
    • getRecords()：获取当前页的记录列表。

    你可以根据需要使用这些信息进行后续处理，比如将查询结果返回给前端或进行其他业务逻辑处理。

请注意，使用 Page 进行分页查询时，需要确保数据库支持分页查询，并在相关的 Mapper 接口中正确配置了分页查询的方法。

以上是使用 Page 进行分页查询的基本步骤，你可以根据具体的需求和代码逻辑进行相应的调整和实现。
