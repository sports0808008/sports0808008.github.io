Maven基本概念

​ 在之前的学习中我们知道如果要开发一个项目，那么肯定会使用到一堆的*.jar包，之前我们并没有发现有任何问题，那是因为我们目前使用*.jar少，如果在以后的中大型项目中至少出现上百个甚至上千个这样的*.jar是很正常的。此时如果在手工的去下载，去管理就是一场噩梦。为了解决这一问题，Maven出现了，只需要在配置文件中进行配置就行了。

​ Maven可以实现项目的打包、编译、对jar包的管理，其中最重要功能是对jar包的管理。

​ maven项目一定有一个pom.xml，我们在相关的pom.xml文件中进行简单的配置之后，就会自动下载我们所需要的jar包。

​ Maven有很多优点，但是也有很多缺点需要你去忍受，比如说创建了项目之后会在项目上出现错误，但是这个错误不会影响你的开发，此时不要去折腾他。

​ 要求你的网络要好，如果网络不好，下到一半的时候出现问题，要删除之前下载的，重新来，甚至要重建项目。

Mybatis的基本概念

     分析之前是怎么去操作数据库，首先要取得Connection接口对象，需要使用DriverManager类取得，之后还需要使用PreparedStatement接口对sql语句进行预编译之后执行，在很早之前还可以使用Statement接口，除此之外还需要使用ResultSet接口取得结果集（就是原始的数据）。

   以上是我们在之前操作数据库需要使用到的接口以及相关的类，如果要带条件进行操作数据，需要使用一堆的setXXX方法设置占位符（“?”）的内容，查询到数据之后还需要使用ResultSet的getXXX方法取得原始的数据，之后再调用vo对象的setXXX方法保存数据，而且以上的设置和取得操作是重复的。

   经过以上的分析发现**传统的****jdbc****开发存在的弊端（复杂和重复代码）**，于是一些框架就出现来解决这一问题，**框架工具将上面的繁杂过程封装了**，这些框架统一叫做**对象映射框架（****orm****）**，将数据表中的数据直接和vo类对象直接对应起来，为用户省去了传统开发中的那一堆复杂的操作。

​ 其中最著名的是Hibernate(全自动的orm框架)和Mybatis（半自动的orm框架，前身是ibatis），使用这些框架将传统的jdbc所使用到的类和接口封装，以及参数的设置和数据的取得封装，我们只需要编写sql语句执行即可，如果要查询数据他会自动返回你要查询的对象

​ ◆Mybatis：是一个轻量级的框架，使用比较灵活（还可以使用注解实现），而且性能比Hibernate高。

​ ◆mybatis-plus:和mybatis进行互补，使用更方便。

​ ◆有部分公司使用的是自己定义的框架，思想都类似

数据库连接池

​ 访问数据库的时候需要取得数据库的连接对象，在之前使用完毕之后就关闭了，如果再需要就从新取得，在这一过程中需要耗费大量的时间，这种方式效率比较低，最好的方式是在项目启动的时候先创建出一定数量的数据库连接对象保存到连接池，需要使用的时候直接从连接池中取得，使用完毕之后在放回连接池，试用这样的方式提高了数据的访问速度，数据库连接池有多中，本次demo中使用mybatis自带的连接池，市面上还有C3P0、DBCP、PROXOOL、DRUID。

SqlSessionFactory 是 MyBatis 的核心接口之一，它用于创建和管理 SqlSession 对象。

SqlSessionFactory 接口定义了一些重要的方法，例如：

  • SqlSession openSession(): 创建一个新的 SqlSession 对象，用于执行数据库操作。
  • SqlSession openSession(boolean autoCommit): 根据指定的参数创建一个新的 SqlSession 对象，并指定是否自动提交事务。
  • SqlSession openSession(ExecutorType executorType): 根据指定的执行器类型创建一个新的 SqlSession 对象。
  • Configuration getConfiguration(): 获取当前 SqlSessionFactory 的配置信息。
  • void close(): 关闭 SqlSessionFactory，释放相关资源。

通过 SqlSessionFactory，你可以创建 SqlSession 实例来执行数据库的增删改查操作。一般来说，SqlSessionFactory 只需要创建一次，然后在应用程序的生命周期内进行重复使用。

创建 SqlSessionFactory 需要一个配置文件（通常是 mybatis-config.xml）或者配置对象来进行相应的配置。具体配置文件的路径以及配置方式，取决于你的项目结构和使用方式。

  • int update(String statement): 执行更新语句，并返回受影响的行数。
