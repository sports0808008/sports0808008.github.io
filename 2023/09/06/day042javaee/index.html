Tomcat服务器的概念与安装

​ 当用户在浏览器浏览一些数据的时候，需要向服务器端发送请求（request），服务器端要对请求做出响应（response），这是客户端和服务器端的一种数据交互，完成这个交互功能的是一个容器，叫做web****容器(*又称为 Servlet 容器) Web 容器负责接收客户端的请求，并将其传递给适当的组件进行处理。它还负责从服务器端获取响应，并将其发送回客户端，web容器有很多种，常见的有*Jboss、Weblogic、Jetty（SpringBoot默认的服务器就是Jetty）、Tomcat（汤姆猫）。

其中 Tomcat 是非常流行和常用的 Web 容器，它是一个轻量级的、开源的 Servlet 容器和 Web 服务器集合体，用于托管 Java Web 应用程序。Spring Boot 默认集成了嵌入式的 Tomcat 作为内置的 Web 容器，因此在开发 Spring Boot 应用时，无需额外安装和配置 Tomcat，可以直接运行应用。当你在浏览器中浏览网页时，浏览器会发送 HTTP 请求到服务器端的 Web 容器，Web 容器将根据请求的 URL 和其他信息来确定应该处理请求的哪个组件（如 Servlet 或控制器），然后组件会生成相应的响应数据，并由容器发送回浏览器，最终在浏览器中呈现出来。这样完成了客户端和服务器端之间的数据交互过程。

对于客户端和服务器端之间的数据交互，涉及到的组件可以是 Servlet 或控制器

Servlet 是 Java Web 应用程序中处理客户端请求和生成响应的组件。它是基于 Java Servlet 技术的 API，用于接收 HTTP 请求、处理请求的数据和逻辑，并生成相应的响应返回给客户端。Servlet 可以运行在 Web 容器中，如 Tomcat、Jetty、JBoss 等。Servlet 主要用于处理底层的请求和响应操作，如获取请求参数、读取请求头、设置响应状态码等，需要开发者编写处理逻辑来完成具体的业务操作。

控制器是 MVC（Model-View-Controller）架构模式中的一部分，用于根据客户端的请求来处理业务逻辑并生成响应。控制器负责接收客户端的请求，调用相应的服务或模型组件进行业务处理，并将处理结果传递给视图组件，最终生成适当的响应返回给客户端。控制器在 Web 应用程序中起到了分发、路由和中央协调的作用，它能够根据请求的路径、类型等信息将请求分派给适当的处理逻辑，并实现请求和响应之间的解耦。控制器常用的技术包括 Spring MVC、ASP.NET MVC、Express.js 等。

无论是 Servlet 还是控制器，它们都是客户端和服务器端之间数据交互的重要组成部分，用于处理请求、执行业务逻辑并生成响应，从而实现客户端和服务器端的交互功能。

什么是HTTP协议（了解）

​ 协议是指计算机通信网络中两台计算机之间进行通信所必须共同遵守的规定或规则，超文本传输协议(HTTP)是一种通信协议，它允许将超文本标记语言(HTML)文档从Web服务器传送到客户端的浏览器。

可以简单理解为电脑之间进行数据的传输要遵循的一些规则。

HTTP协议是无状态的（重点概念）

​ http协议是无状态的，同一个客户端的这次请求和上次请求是没有对应关系，对http服务器来说，它并不知道这两个请求来自同一个客户端。 为了解决这个问题， Web程序引入了Cookie机制来维护状态.（https://jingyan.baidu.com/article/fedf073780da6335ac897799.html）

​ 和无状态概念对应的就是有状态。你的微信登陆之后和腾讯的服务器是一直保持连接状态（直到你的退出登陆之后才能断开连接）

​ 无状态的又被叫做 B/S架构 有状态的被叫做C/S架构。

Web服务器，浏览器（重点概念）

​ 当我们打开浏览器，在地址栏中输入URL，然后我们就看到了网页。 原理是怎样的呢？

​ 实际上我们输入URL后，我们的浏览器给Web服务器发送了一个Request（请求）, Web服务器接到Request后进行处理，生成相应的Response（响应），然后发送给浏览器， 浏览器解析Response中的数据,这样我们就看到了网页

URL详解（掌握）

url就是浏览器地址栏的信息

https://www.tmall.com/

URL(Uniform Resource Locator) 地址用于描述一个网络上的资源的地址, 基本格式如下

schema://host[:port#]/path/…/[?query-string]

scheme 指定低层使用的协议(例如：http, https, ftp)

host HTTP服务器的IP地址或者域名（本机就是localhost或者127.0.0.1）

port# HTTP服务器的默认端口是80，这种情况下端口号可以省略。如果使用了别的端口，必须指明，例如 http://www.cnblogs.com:8080/

path 访问资源的路径

query-string 发送给Web服务器的数据，数据可以是查询的条件，也可以是保存到数据库的内容

URL 的一个例子

http://www.mywebsite.com/sj/test/test.aspx?name=sviergn**&**x=true
Schema: http
host: www.mywebsite.com
path: /sj/test/test.aspx
Query String: name=sviergn&x=true

状态码（了解）

Response 消息中的第一行叫做状态行，由HTTP协议版本号， 状态码， 状态消息 三部分组成。

状态码用来告诉HTTP客户端,HTTP服务器是否产生了预期的Response.

HTTP/1.1中定义了5类状态码， 状态码由三位数字组成，第一个数字定义了响应的类别

1XX 提示信息 - 表示请求已被成功接收，继续处理

2XX 成功 - 表示请求已被成功接收，理解，接受

【比如说200表示请求数据成功】

3XX 重定向 - 要完成请求必须进行更进一步的处理

【请求发送到服务器端时候会做一次或者多次的重新定向请求】

4XX 客户端错误 - 请求有语法错误或请求无法实现

【404 就是客户端请求的资源不存在，404 检查后端提供的资源的路径 与你在浏览器输入的是否一致】

5XX 服务器端错误 - 服务器未能实现合法的请求

【比如说500表示服务器端出现了异常】

看看一些常见的状态码

200 OK

最常见的就是成功响应状态码200了， 这表明该请求被成功地完成，所请求的资源发送回客户端

302 Found

重定向，新的URL会在response 中的Location中返回，浏览器将会自动使用新的URL发出新的Request

例如在IE中输入， http://www.google.com. HTTP服务器会返回302， IE取到Response中Location header的新URL, 又重新发送了一个Request.

304 Not Modified

代表上次的文档已经被缓存了， 还可以继续使用，

例如打开博客园首页, 发现很多Response 的status code 都是304

400 Bad Request 客户端请求与语法错误，不能被服务器所理解

403 Forbidden 服务器收到请求，但是拒绝提供服务（比如说服务器端仅仅提供了处理get请求的方法，但是你发送的请求是post请求）

404 Not Found ：请求资源不存在（输错了URL）【以后出现了404一定是路径不对】

500 Internal Server Error 服务器发生了不可预期的错误

503 Server Unavailable 服务器当前不能处理客户端的请求，一段时间后可能恢复正常

Servlet概念与web项目构建

Servlet概念

我们安装了Tomcat，tomcat是一个容器（又叫做上下文），这个容器可以处理用户发送的请求（Request），之后会为请求做出一个响应（Response），此时用户只能请求一些现成的资源（比如安装成功之后看到的tomcat首页），容器直接返回给客户，如果要容器能接受用户传递的参数（比如登录的时候将用户名密码、验证码提交给服务端的时候）那么还需要在容器有专门的程序实现这样的数据交互，这样的程序叫做服务器端小程序，该程序被叫做servlet。要实现servlet的编写需要继承一个抽象类“HttpServlet”， 该类可以通过添加相关依赖来导入。
