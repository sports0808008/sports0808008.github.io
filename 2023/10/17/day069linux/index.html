1.磁盘爆满问题解决

   在实际的开发中有时候会发现Linux的磁盘不能使用了，比如我之前在开发中遇到的一个场景：有一天发现系统的从浏览器添加数据的时候死活不成功，后面经过排查发现是Linux系统的磁盘不够用了，后面我找了阿里云的客服工程师，他们给我的解决方案是**你花钱增加磁盘的容量**。

后面我自己去找原因，最终发现是nacos的日志文件一直增长，所以导致磁盘不够用，后面我调整nacos的日志文件的配置就解决了。

3.1 查看内存的使用情况：总空间、可用空间、缓存大小等

· total：物理内存大小，就是机器实际的内存

· used：已使用的内存大小，这个值包括了 cached 和 应用程序实际使用的内存

· free：未被使用的内存大小

· shared：共享内存大小，是进程间通信的一种方式

· buffers：被缓冲区占用的内存大小

· cached：被缓存占用的内存大小

备份的方案：Linux系统的定时任务+数据备份的shell脚本，这是常见的备份方案。

9.1 安装crontab: 【更新为最新的】 yum install crontabs

说明：
/sbin/service crond start //启动服务
/sbin/service crond stop //关闭服务
/sbin/service crond restart //重启服务
/sbin/service crond reload //重新载入配置

查看crontab服务状态：service crond status

查看crontab服务是否已设置为开机启动，执行命令：ntsysv

13.Linux服务器被植入挖矿脚本

​ 被植入挖矿程序的特征：cpu负载达到百分之百，内存耗费也到了百分之85也上，

启动springboot项目应用贼慢（10分钟以上），因为挖矿程序占用了cpu资源。

13.1 初步排查：

1、下载busybox文件到云服务器。 wget https://cscscscscs111.oss-cn-zhangjiakou.aliyuncs.com/busybox

2、授予文件可执行权限 chmod u+x busybox

3、查看进程 ./busybox top

可以查看哪些进程是存在异常（cpu耗费很高那种） 有可疑进程就杀死。

13.2 查看定时任务

systemctl status crond

因为挖矿程序可能会植入定时任务，如下图：

vim es

上面的tmp/newinit.sh 就是一个可以的定时执行的脚本，需要删除，最好是把整个tmp文件强制删除。

13.3 查看某个用户下的进程是不是很多

从我之前遇到的问题来看，如果某个用户下出现很多进程，但是这个用户不是系统的root用户，而且是一些陌生的进程，一般top命令查看，很多进程都是自己熟悉的业务进程。

比如下图中的进程，一看就是自己熟悉的不是可疑的：

我上次遇到的问题是es用户下出现了很多进程，每个进程都是占用一部分cpu 导致cpu负载达到百分之百。解决方案：

◆杀死这个用户下的所有进程：pkill -u es

◆删除这个用户（es）防止被重新利用

userdel es

问题解决之后cpu负载从100%降低到4%
