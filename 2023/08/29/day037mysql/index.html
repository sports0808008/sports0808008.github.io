WHERE子查询

思考上节课的课后作业：查询出薪资低于平均工资的雇员信息

子查询就是将一个查询的结果作为另外一个查询（主查询）的数据来源或者是判断条件的的查询，常见的子查询有WHERE子查询、FROM子查询、SELECT子查询、HAVING子查询、EXISTS子查询，子查询必须使用小括号括起来。

本次课学习的是WHERE子查询，就是在WHERE子句中继续使用查询来实现一些需求。

查询出低于平均工资的雇员信息

查询出平均薪资 SELECT AVG(sal) FROM emp
SELECT * FROM emp WHERE sal<( SELECT AVG(sal) FROM emp );

· SELECT AVG(sal) FROM emp; [返回的结果集是单行单列类型]

· 子查询需要用小括号括起来

查询出公司最早入职的雇员信息

SELECT * FROM emp WHERE hiredate=( SELECT MIN(hiredate) FROM emp );

查询出与scott从事同一工作，并且工资相同的雇员的信息

第一步：查询出scott的工作职位

SELECT job FROM emp WHERE ename=’scott’;

第二步：查询scott的薪资

SELECT sal FROM emp WHERE ename=’scott’;

第三步：将以上两个查询的结果作为条件（子查询）

SELECT * FROM emp WHERE job=(SELECT job FROM emp WHERE ename=’scott’) AND sal=(SELECT sal FROM emp WHERE ename=’scott’) AND ename<>‘scott’;

第二种方式：

SELECT * FROM emp WHERE (job,sal)=(SELECT job,sal FROM emp WHERE ename=’scott’) AND ename<>‘scott’;

·主查询中的条件判断字段的顺序要子查询中需要查询的字段的顺序保持一致

· 子查询的返回值是单行多列的结果集

查询出与 职位是经理的雇员的薪资相同的员工信息

第一步：先查询出经理职位的雇员的所有薪资【结果集类型是多行单列类型】

SELECT sal FROM emp WHERE job=’manager’;

第二步：使用子查询

SELECT * FROM emp WHERE sal IN( SELECT sal FROM emp WHERE job= ‘manager’ ) AND job<>‘manager’;

返回的结果集是多行单列的结果集。

查询出薪资比职位是经理的雇员中最低薪资高的员工信息

方式一：

SELECT * FROM emp WHERE sal>( SELECT MIN(sal) FROM emp WHERE job=’manager’ );

方式二：使用any实现

SELECT * FROM emp WHERE sal>ANY( SELECT sal FROM emp WHERE job=’manager’ );

**sal>any:**其实表示的是sal大于子查中的最小的一个值即可

查询出薪资比职位是经理职的雇员中最高薪资低的员工信息

SELECT * FROM emp WHERE sal<(SELECT MAX(sal) FROM emp WHERE job=’manager’);
SELECT * FROM emp WHERE sal<ANY( SELECT sal FROM emp WHERE job= ‘manager’ );

**sal<any:**其实表示的是sal小于子查中最大的一个值即可

查询出薪资比 职位是经理的雇员中最低工资低的员工的信息

SELECT * FROM emp WHERE sal<( SELECT MIN(sal) FROM emp WHERE job= ‘manager’ );
SELECT * FROM emp WHERE sal< ALL(SELECT sal FROM emp WHERE job= ‘manager’);

sal<all: 表示薪资小于子查询中的所有即可。

查询出比 职位是经理的雇员中最高工资 高的员工的信息

SELECT * FROM emp WHERE sal>(SELECT MAX(sal) FROM emp WHERE job=’manager’ );
SELECT * FROM emp WHERE sal>ALL( SELECT sal FROM emp WHERE job=’manager’ );

sal>all: 表示的薪资大于子查询中的所有结果就行。

HAVING子查询

HAVING子句是对分组统计函数进行过滤的子句，也可以在HAVING子句中使用子查询，这就是HAVING子查询。

查询出 平均薪资 高于 所有员工平均工资的 职位的名称、以及职位的人数、职位的平均工资

第一步：查询出所有雇员的平均工资

SELECT AVG(sal) FROM emp

第二步：按照职位分组，使用子查询

SELECT job,COUNT(*) 人数,AVG(sal) FROM emp GROUP BY job HAVING AVG(sal)>(SELECT AVG(sal) FROM emp);

查询出 平均薪资 最高 的职位 名称和该职位的平均工资。

第一步：查询出平均薪资最高的职位的平均薪资

查询出每个职位的平均薪资 SELECT AVG(sal) FROM emp GROUP BY job 从所有职位的平均薪资中查询出最高的平均薪资 SELECT MAX(avgsal) FROM (SELECT AVG(sal) avgsal FROM emp GROUP BY job) AS temp;

第二步：使用子查询

SELECT job,AVG(sal) FROM emp GROUP BY job HAVING AVG(sal)=(SELECT MAX(avgsal) FROM (SELECT AVG(sal) avgsal FROM emp GROUP BY job) AS temp);

查询出薪资排名第三的雇员的信息（不考虑并列薪资）

第一步：查询第一名薪资

SELECT MAX(sal) FROM emp;

第二步：第二名薪资

SELECT MAX(sal) FROM emp WHERE sal<(SELECT MAX(sal) FROM emp);

第三步：第三名薪资

SELECT MAX(sal) FROM emp WHERE sal<( SELECT MAX(sal) FROM emp WHERE sal<(SELECT MAX(sal) FROM emp));

第四步：查询我们需要的信息即可

SELECT * FROM emp WHERE sal=( SELECT MAX(sal) FROM emp WHERE sal<( SELECT MAX(sal) FROM emp WHERE sal<(SELECT MAX(sal) FROM emp)));

这有点像去年一个学长去面试（新东方）：有十万学生的信息，查询出成绩第三名的学生的信息。

但是以上的实现方式如果是前面名可以考虑使用，如果是查询第100名呢？可以考虑排序再分页实现，如果有成绩并列的情况需要结合程序处理。

FROM子查询

在之前我们在讲解FROM子句的时候一直在FROM子句之后跟上的是数据来源而没有说是数据表，其实上FROM子句的数据来源不仅仅可以是数据表，还可以是一个查询的结果（一般是多行多列类型的数据），把一个查询的结果作为主查询的数据来源，这就是FROM子查询。

查询出平均工资高于2000的职位名称以及该职位的平均工资

第一种方式：可以使用HAVING 子句实现

SELECT job,AVG(sal) FROM emp GROUP BY job HAVING AVG(sal)>2000;

第二种方式：使用FROM子查询实现

SELECT job,avgsal FROM ( SELECT job,AVG(sal) AS avgsal FROM emp GROUP BY job ) AS temp WHERE avgsal>2000;

总结

1、FROM子查询的结果作为主查询的数据来源，子查询的结果相当于一张临时数据表

2、必须为子查询定义别名

3、如果要使用子查询中的字段则应该使用“别名.字段名”的方式进行引用（在oracle必须这样写，在mysql中可以不写）。

EXISTS子查询（了解）

EXISTS子查询的特征：将主查询中的数据带到子查询中进行验证，如果验证成功则被验证的数据就显示， 反之不显示。

查询出有部门的雇员的信息

SELECT * FROM emp WHERE deptno IS NOT NULL;
SELECT * FROM emp WHERE deptno IN( SELECT deptno FROM dept );
【使用EXISTS子查询实现】 SELECT * FROM emp e WHERE EXISTS( SELECT * FROM dept d WHERE e.deptno=d.deptno );

查询出还没有部门的 雇员的 雇员的信息

SELECT * FROM emp e WHERE NOT EXISTS( SELECT * FROM dept d WHERE e.deptno=d.deptno );

使用NOT EXISTS的时候也是将主查询中的数据带入子查询进行验证，如果验证成功则不显示验证的数据，反之则显示数据。

EXISTS子查询在后面的作业中会出现，本节课我们只是举一个简单的例子来说明了Exists子查询的流程。

分页查询（重点）

分页查询是百分之九十九的项目都要使用到的，比如说如果数据很多但是要显示到浏览器页面，这种情况下如果所有的数据都使用一个页面显示，那是不合理的，而是要使用分页显示。

要实现分页需要知道两个参数：

◆currentPage:表示当前页（cp）

◆lineSize:表示每页显示的数据量（ls）

mysql中的分页查询

如果要在mysql中实现分页查询需要使用到一个关键字LIMIT ，关键字之后需要两个参数 m（从第m条开始查询）,n(往后查询n条数据) ，最终的实现就是:

SELECT * FROM 数据来源 WHERE 过滤条件 GROUP BY 分组字段 HAVING 对统计函数进行过滤 ORDER BY 排序字段 排序的方式 LIMIT m,n

m的含义表示从数据库的第m（对应参数（cp-1）*ls）条开始查询（mysql中的第一条数据m==0）

n的含义是从第m条数据开始往后查询n（对应参数ls）条数据。

◆查询第1页数据，每页显示3条数据

SELECT * FROM emp LIMIT 0,3;

·查询第1页数据，每页显示4条数据

SELECT * FROM emp LIMIT 0,4;

·查询第2页数据，每页显示5条数据（m=(2-1)*5==5）

SELECT * FROM emp LIMIT 5,5;

·查询第3页数据，每页显示6（ls==6）条

SELECT * FROM emp LIMIT 12,6;

总结

通过以上的demo可以总结公式：

m=(cp-1)*ls

n=ls

以上是mysql的分页实现，你一定要非常的熟悉这种计算公式，到后面就不讲了，直接使用。

约束的概念

上节课发现了最后一条数据的编号为null，我们说过数据表必须有一个字段表示主键作为数据的唯一标识，这种字段中的数据不应该为null，此时在创建数据表的时候可以指定该字段不能为null（给该字段做一个约束）

指定字段不能为空

– 删除数据表 DROP TABLE IF EXISTS teacher; – 创建数据表 CREATE TABLE teacher( tno INT(4) NOT NULL, – 老师编号,约束该字段必须有值 tname VARCHAR(10), – 老师姓名 age INT(3), – 年龄 hiredate DATE – 入职日期 最后的一个字段不需要加逗号 ); – 插入数据 INSERT INTO teacher VALUES(1001,’王老师’,18,now()); INSERT INTO teacher VALUES(1002,’李老师’,18,now()); INSERT INTO teacher VALUES(1003,’李老师’,18,now()); INSERT INTO teacher(tname,age,hiredate) VALUES(‘张老师’,23,now())

唯一约束

– 删除数据表 DROP TABLE IF EXISTS teacher; – 创建数据表 CREATE TABLE teacher( tno INT(4) NOT NULL UNIQUE, – 老师编号,约束该字段必须有值 tname VARCHAR(10), – 老师姓名 age INT(3), – 年龄 hiredate DATE – 入职日期 最后的一个字段不需要加逗号 ); – 插入数据 INSERT INTO teacher VALUES(1001,’王老师’,18,now()); INSERT INTO teacher VALUES(1002,’李老师’,18,now()); INSERT INTO teacher VALUES(1003,’凤雏老师’,18,now()); INSERT INTO teacher VALUES(1003,’张老师’,18,now());

索引的概念（了解）

什么是索引

数据库索引指的是数据库管理系统中一个排序的数据结构，以协助快速查询、更新数据库表中的数据。简单理解就是类似书籍的目录，独立于书籍内容之外，能够快速根据目录查到到需要的文章的页码。如果没有目录，我们就要一页一页地翻页来查找文章，同理，如果没有索引的话，数据库系统只能一行一行地查找数据，注意，如果没有达到限制查找的行数，这样的遍历就要一直进行下去，直到全部遍历完。

显而易见，这样的查找效率是不可接受的。所以我们需要索引的帮助，索引可以提升数据查找的效率，从而大大提升数据库的性能。

数据结构

学习MySQL索引的第一步，是先了解下MySQL索引的数据结构，也就是索引是用什么结构来存储的，了解了索引的数据结构才能帮助我们更好地理解，为什么索引有那么高效的检索性能，以及怎么样高效使用索引，MySQL索引常见的数据存储结构有B+树结构

B+树结构

下面再来介绍下B+树结构的索引，在介绍前，我们先来了解下可以用来其他快速查找数据的树结构，以便理解为什么要用B+树。

平衡二叉排序树

平衡二叉排序树是基于二分法的策略提高数据的查找速度的二叉树的数据结构，其树高平衡，不会出现单链的情况。其结构如下所示：

平衡二叉排序树查找数据的速度近于二分法查找O(logn)，然而二叉树搜索需要很高的I/O次数，导致性能并不如人意。这是因为每个节点存储的数据比较少，每次节点进行比较的时候都需要进行一次I/O获取节点的数据，而在操作系统中，I/O的成本是很高的。为了解决I/O次数高的问题，我们就有了B树。

B树

B树又名平衡多路查找树，顾名词义，它的查找路径不止两个，也就是不止两个子节点。其结构如下：

在MySQL中，B树（B-Tree）和B+树（B+Tree）是两种常见的索引结构，用于优化数据库的数据存储和查询性能。下面详细分析MySQL中的B树和B+树的区别：

节点结构：

B树：MySQL中的B树每个节点既包含索引键，也包含对应的数据。叶子节点和内部节点的结构是一致的，都包含了键和数据。

B+树：MySQL中的B+树的内部节点只包含索引键，而数据只存储在叶子节点上。内部节点的主要作用是导航到叶子节点，而叶子节点通过链表连接形成有序的数据序列。

叶子节点结构：

B树：B树的叶子节点包含了索引键和对应的数据，叶子节点之间没有特殊的连接方式。

B+树：B+树的叶子节点只包含索引键和数据指针，叶子节点之间通过链表连接形成一个有序的链表结构。这使得范围查询和顺序遍历更加高效。

指针数量：

B树：B树的每个节点包含指向子节点的指针，因此节点中存储了数据和指针，可能会限制节点中存放的键值对数量。

B+树：B+树的内部节点只包含指向子节点的指针，不存储键值对。这意味着内部节点可以容纳更多的指针，从而减少树的高度，提高查询性能。

查询性能：

B树：B树由于在每个节点都存储数据，因此在某些情况下，单次查找可能更快。但在范围查询和顺序遍历等操作时，B树可能会涉及更多的磁盘I/O。

B+树：B+树的叶子节点形成有序链表，这对于范围查询和顺序遍历操作非常有利。同时，由于内部节点没有存储数据，磁盘I/O更加高效，通常在数据库索引中具有更好的性能。

B-tree的优缺点

B树的优点是每层的节点数增加了，树高也大大减少，这就大大减少了I/O次数，读取一个节点的时候可以一次性读取大量关键字数据。 但还是存在以下问题：

1.每个节点中既要存索引信息，又要存其对应的数据，如果数据很大，那么当树的体量很大时，每次读到内存中的树的信息就会不太够。

2.B树遍历整个树的过程和二叉树本质上是一样的，B树相对二叉树虽然提高了磁盘IO性能，但并没有解决遍历元素效率低下的问题。

为了解决这些问题，也就有了我们今天的主角：B+树

B+树

B+树是B树的升级版，更充分地利用了节点的空间。其结构如下：

和B树的主要区别如下：

◆内部节点不保存数据，只是用来索引，这样非叶子节点的所占的内存空间就变小了，每次读到内存中的索引信息就会更多一些，相当于减少了磁盘IO次数。

◆数据都是存在叶子节点，这样保证了相近的数据都能存在同一块数据块里。另外也使得B+树的查询次数更稳定，每次查询次数都是相同的，需要查询到叶子节点

◆叶子节点的指针指向下一个数据对应的叶子节点，因此B+树具备了天然排序功能，在排序和范围查找的时候更方便，可以通过叶子节点的指针找到下一个叶子节点的位置

◆B+树可以方便地做全表搜索，只需要从第一个叶子节点顺序往后面扫描即可，而B树则需要做树的遍历。

观察代码

在EXPLAIN的输出中，关注以下列：

type：表示查询的访问方式，常见的有”ALL”、”index”、”range”等，”index”和”range”表示使用了索引。

key：显示哪个索引被使用。

possible_keys：显示可能使用的索引列表。

Extra：额外的信息，可能显示”Using index”。

EXPLAIN SELECT * FROM emp WHERE sal>1000;
CREATE INDEX 索引名 ON 数据表(字段名)

相当于将数据表的指定字段的值保存到索引树节点中。

创建索引

– 在emp表中的sal字段上创建索引 CREATE INDEX emp_sal_index ON emp(sal);

现在创建了索引，查询的时候就在索引范围内扫面，就提高了查询的效率。

删除索引

– 删除索引 ORACLE: DROP INDEX emp_sal_index; MYSQL: ALTER TABLE table_name DROP INDEX index_name;

以上是一个索引作用于一个字段，还可以一个索引作用于多个字段，这样的索引叫做复合索引。

复合索引创建

复合索引就是一个索引在多个字段上创建，就是一个索引作用于多个字段。

– 创建复合索引 CREATE INDEX emp_job_sal_index ON emp(job,sal);
