|-从数据库中取得数据到程序中：将utf-8的数据解码成字节码，之后再编码成gbk,再读取到程序中。

·如果是在windows系统上运行程序一般不会出现乱码问题，但是如果将windows上的项目移植到Linux上就可能出现中文乱码现象。

事务处理

数据的更新是一个很常见的需求，而且在数据的更新的时候很可能会

要求一批操作中要么都成功要么都失败，这种特性叫做事务的原子性，稍后我们用转账业务来说明这一问题。

补充：所谓的事务就是一次对数据库中数据的操作（可以发送多条sql语句指令），一般从取得连接开始—>执行sql语句—>关闭连接，这个过程就是一个事务的生命周期。

简单的说就是一次对数据库中数据的操作就是事务，保证原子性是事务的要求之一。

SMITH转账给ALLEN 100元出现的问题（观察问题）

​ 出现了smith账户的减少100，但是allen的账户没有收到，出现这个现象的原因是事务没有保证原子性，上面的demo中的事务是自动提交的（就是自动把sql语句完全执行）。

​ 如果要解决这个问题的思路：可以将两条sql语句指令先预存到缓存中，之后判断所有的sql语句指令，如果都正确则再全部执行预存起来的sql，如果出现了一条sql指令错误，则全部都撤销。

SMITH转账给ALLEN 100元正确做法

​ 以上的demo中第一个sql正常执行成功，但是第二个sql执行失败，此时我们先取消了事务的自动提交，只有所有的sql都执行成功再进行手工提交事务，否则就事务回滚，保证了事务的原子性（要么就全部成功，要么全部失败）。

总结

  1. 事务的原子性的概念（一批操作中要么全部失败要么都成功）
  2. 事务的概念（就是对数据库中数据的一次操作过程就校叫做一次事务）
  3. 程序中的事务提交和回滚的业务你要熟悉。

数据查询

​ 之前我们使用“PreparedStatement”将sql先编译成指令，再发送给数据库执行，如果发送的是一条查询数据的sql语句，执行之后会将查询到数据返回，查询到的数据会被封装到接口“Resultset”对象中返回，要在程序中取得查询的数据就需要使用到“ResultSet”接口。

​ 最终将数据封装到一个“ResultSet”接口对象中，该对象叫做结果集对象（就是查询到的结果的集合）

​ A table of data representing a database result set, which is usually generated by executing a statement that queries the database.

​ A ResultSet object maintains a cursor pointing to its current row of data. Initially the cursor is positioned before the first row. The next method moves the cursor to the next row, and because it returns false when there are no more rows in the ResultSet object
