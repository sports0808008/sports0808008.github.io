分析三张数据表结构

如果是oracle数据库则安装的过程中选择了“生成示例方案”则会默认提供四张数据表供给学习，其中三张数据表非常的熟悉（emp、dept、salgrade）

但是现在我们是用的是mysql，所以默认没有这几张数据表，为了方便我们学习，所以提供了这几张数据表的脚本，直接复制运行即可。

观察问题

SELECT * FROM emp;

在编写SQL语句的时候要注意一个规则：关键字使用大写字母，其他非关键字使用小写字母（本来在sql语句中是不区分大小写的）

第一张数据表（emp）

表名：emp，该数据表是用来保存雇员的信息

可以使用“DESC 表名”的方式查询出数据表的结构

字段名 字段类型 字段功能
EMPNO int(4) 保存雇员的编号，只能保存数字类型的数据，最多保存4位数字
ENAME VARCHAR(10) 保存雇员的名字，保存字符串类型数据，最多保存10个英文字符（在oracle中三个英文字符等于一个中文）
JOB VARCHAR(10) 保存雇员的职位，保存的数据类型字符串，最多保存10个英文字符
MGR int(4) 保存的是雇员的领导的编号，类型是数字，最多可以保存4位数字
HIREDATE DATE 保存的是雇员的入职日期，类型是日期类型
SAL double(7,2) 保存雇员的薪资，类型是数字类型，7表示最大的可以保存7位数字，2表示最大小数位是2，整数位最大是7-2=5位
COMM double(7,2) 保存雇员的佣金，类型是数字类型，7表示最大的可以保存7位数字，2表示最大小数位是2，整数位最大是7-2=5位
DEPTNO int(2) 保存雇员所在部门的编号，类型是数字，最大保存2位数字

注意：每一张数据表都有一个非常重要的列，表示数据的编号，一般情况这个列叫做主键，它是数据的唯一标识。

第二张数据表（dept）

名称：dept，该数据表保存的是部门信息

字段名 字段类型 字段功能
DEPTNO int(4) 保存部门的编号，类型是数字，最大保存4位数字
DNAME VARCHAR(30) 保存部门的名称，类型是字符串，最大保存30个英文字符
LOC VARCHAR(30) 保存的是部门所在的位置，类型是字符串，最大保存30个英文字符

第三张数据表（salgrade）

名称：salgrade，保存的是薪资等级信息

字段名 字段类型 字段功能
GREADE int(4) 保存的是薪资等级信息
LOSAL double(7,2) 保存 的是薪资等级的最低薪资
HISAL double(7,2) 保存 的是薪资等级的最高薪资

补充：mysql中的字符串类型不是String（java中的），而是varchar(字符数)类型

总结：

1、简单命令的使用（SELECT * FROM 数据表）

2、数据表的结构，这几张数据表在接下来的几天都要使用到，所以你要现在就记下来。

3、主要研究的是查询，查询也是数据库学习的难点和重点，其他东西都是辅助。

简单查询

数据库有一门专门属于自己的语言，该语言的核心就是查询语句，难点也是查询、重点也是查询、使用最多的也是查询，这门们语言叫做“Structured Query Language” 结构化查询语言,就是常说的sql语句。

在学习简单查询之前我们先了解SQL语句的分类。

SQL分类（了解）

·数据操纵语言（DML Data Manipulation Language ）:

SELECT（查询） INSERT（插入数据） UPDATE（更新数据） DELETE（删除数据）

·数据定义语言（DDL Data definition language ）:

CREATE ALTER（修改数据表的结构） DROP RENAME TRUNCATE

·数据控制语言（DCL Data Control Language ）:

GRANT （比如说授权远程连接、权限授权等等）

·事物控制语言 Transaction commit rollback

所谓的简单查询就是没有任何限定条件的查询，但是在开发中一般不会使用到简单查询。

基本的语法如下：

·第一种形式

② SELECT * 【SELECT 子句】
① FROM 数据来源（可以是数据表） 【FROM 子句】

|- 以上的两个子句在数据库中执行指令的时候是有顺序的，先执行FROM子句，再执行SELECT子句。

|-*表示查询所有的字段信息，但是在开发中尽量保证需要什么字段就指定什么字段，不要星号，但是在课程讲解中为了方便我就很多时候会直接使用星号。

·第二种形式：

② SELECT 字段名,字段名,字段名… 【SELECT 子句】
① FROM 数据来源 【FROM子句】

这种形式查询的内容是指定的，不是*，需要哪些字段信息就指定出来。

（在实际的应用中，一般使用第二种形式，但是在课程的讲解中为了方便我会经常使用到第一种形式。）

查询emp的全部列（字段）的内容

SELECT * FROM emp;

查询每个员工的编号、姓名、职位、工资

SELECT empno,ename,job,sal FROM emp;

查询job列（字段）的内容

SELECT job FROM emp;

以上查出了职位的信息，但是以上的查询结果有重复的，如果要去掉重复的信息可以使用关键字 ‘’DISTINCT”

解决重复的数据

SELECT DISTINCT job FROM emp;

使用DISTINCT要注意的问题

SELECT DISTINCT job,sal FROM emp;

使用DISTINCT 只有查询的所有字段的内容都相同才会认为是重复的。

查询每个雇员的编号、姓名、基本年薪

SELECT empno,ename,sal*12 FROM emp;

发现了可以在sql语句中对数字类型的字段使用四则运算。

但是查询出的年薪显示的时候是“SAL*12”，这种显示不好，应该使用英文单词来表示年薪，如果要实现这样的需要 则要使用到别名查询。

别名使用查询

SELECT empno AS 编号,ename AS 姓名,job AS 职位,sal*12 AS 年薪 FROM emp; SELECT empno 编号,ename 姓名,job 职位 ,sal*12 年薪 FROM emp;

以上发现了可以在SELECT子句中为查询的字段起别名，而且AS可以省略。

查询员工的编号、姓名、年薪（有年终奖10000、高温补贴1000）

SELECT empno,ename,sal*12+10000+1000 AS income FROM emp;

总结

  1. 基本查询的使用 分为SELECT 字句 和FROM 字句。
  2. 在数据库中执行的时候是先执行FROM字句 再执行SELECT字句。

限定查询

之前学习的查询都是简单查询，简单查询就是查询一张数表中的数据而且没有限定条件，在开发中都是有限定条件查询，如果开发一些比较敏感的系统你没有限定查询，你有承担承担法律责任的风险。

限定查询的基本语法：

③ SELECT * 【SELECT 子句，作用就是显示数据】
① FROM 数据来源,数据来源… 【FROM 子句，找数据源】
② WHERE 过滤条件 【WHERE 子句，筛选出满足条件的数据】

WHERE 子句是对数据进行条件判断，之后选择出满足条件的数据。

查询基本工资高于1500的雇员的信息

SELECT * FROM emp WHERE sal>1500;

查询smith的完整信息

SELECT * FROM emp WHERE ename=’smith’;

查询职位不是销售人员的雇员编号、姓名、职位
SELECT empno,ename,job FROM emp WHERE job <>‘salesman’; 【第二种写法】

查询工资在1500~3000之间的雇员信息

SELECT * FROM emp WHERE sal>=1500 AND sal<=3000;
SELECT * FROM emp WHERE sal BETWEEN 1500 AND 3000;

使用BETWEEN AND 的时候小的数字或者日期放到AND 的前面，大的一个放到AND后面。

以上的两种方式使用第二种会更好，第一种会被数据库认为是两个判断条件，第二种（BETWEEN AND）则会被认为是一个判断条件，所以第二种效率高。

查询工资大于2000或者职位是办事员的信息

SELECT * FROM emp WHERE sal>2000 OR job=’clerck’ ; 【第一种写法】
SELECT * FROM emp WHERE job=’clerck’ UNION ALL SELECT * FROM emp WHERE sal>2000;

使用UNION ALL 可以将两个查询的结果并到一起显示（但是两个查询的关系只能是或者[OR]关系），一般用来代替OR条件查询，这样的好处是避免索引失效（后面的内容）。

查询所有在1981年入职的雇员的信息

SELECT * FROM emp WHERE hiredate>=’1981-01-01’ AND hiredate<=’1981-12-31’; 【第一种】
SELECT * FROM emp WHERE hiredate BETWEEN ‘1981-01-01’ AND ‘1981-12-31’; 【第二种】

查询所有领取佣金的雇员信息

查询出佣金不为null的雇员的信息

SELECT * FROM emp WHERE comm IS NOT NULL;

查询没有佣金的雇员的信息

SELECT * FROM emp WHERE comm IS NULL;

查询出编号是7369,7765,7788的员工的信息

SELECT * FROM emp WHERE empno=7369 OR empno=7765 OR empno=7788; 【第一种】
SELECT * FROM emp WHERE empno IN(7369,7765,7788);

查询雇员编号不是7788,7521,7499的员工信息

SELECT * FROM emp WHERE empno<>7788 AND empno<>7521 AND empno<>7499;
SELECT * FROM emp WHERE empno NOT IN (7788,7521,7499);

总结：

  1. 上面的所有查询你都要掌握
  2. 各字句的执行顺序 ① from字句 ② where字句 ③ select字句。

多表查询（第1讲）

如果需要查询雇员的信息，信息包括编号、姓名、薪资、所在部门的名称、所在部门的位置。

如果要从多张数据表中查询数据，那么就要使用到多表查询，多表查询又叫做连接查询，在之前我们学习FROM子句的时候说了数据来源可以有多个，那么多表查询就需要有多个数据来源。

③SELECT * 【SELECT子句】
① FROM 数据来源,数据来源… 【FROM子句】
② WHERE 过滤条件 【WHERE子句】
④ ORDER BY 排序的字段 ASC|DESC 【ORDER BY 子句】

统计emp表中的数据量

可以使用数据库提供的一个统计函数（COUNT(*|字段)）

SELECT COUNT(*) 数据量 FROM emp;

统计dept表中的数据量

SELECT COUNT(*) FROM dept;

实现多表查询

SELECT * FROM emp,dept;

其实上我们查询了两张数据表，本质的目的是查询雇员的信息以及该雇员的部门信息（这是需求），而且每个雇员的信息只查询一次。

但是查询出了56条数据，其中emp表中有14条数据，dept中有4条数据，4*14=56，最终多表查询出的数据量就是两张表中的数据量的乘积。这样的数据很显然是存在重复的，这种现象叫做笛卡尔积现象，如何解决笛卡尔积现象呢？只需要在查询的时候需要消除笛卡尔积，在查询中增加一个WHERE子句，WHERE子句的过滤条件是两张数据表中的关联字段相等。

消除两张数据表的笛卡尔积

SELECT * FROM emp,dept WHERE emp.deptno=dept.deptno;

以上的方式只是在显示数据的时候去掉了笛卡尔积的重复数据，在数据库执行查询指令的时候本质上是没有消除笛卡尔积（已经查询出了56条数据，只不过使用WHERE 子句过滤出满足条件的数据），所以在开发中有一个规则：数据量比较大的时候不要使用连接查询（多表查询），因为会产生很大的笛卡尔积导致性能耗费，但是数据量小的时候可以使用多表查询。

别名的使用

| SELECT empno,ename,job,sal,d.deptno,dname,loc FROM emp AS e,dept AS d WHERE e.deptno=d.deptno; |

别名的作用是简化sql的复杂 ，让其在引用的时候变得很简洁。也发现了能在WHERE子句中引用FROM子句中定义的别名，证明了WHERE子句在FROM子句的后面执行。

总结：

1、多表查询就是从多张数据表中查询数据

2、多表查询又叫做连接查询

3、多表查询会产生笛卡尔积，在数据量大的时候不要使用多表查询（数据量小的时候可以使用）。

连接查询(第2讲)

连接查询就是多张数据表进行连接然后查询数据（也就是我们之前学习的多表查询），上节课我们学习的多表查询就是连接查询，连接查询又分为内连接查询和外连接查询

内连接查询：只有满足条件的数据才会被显示

外连接查询：可以控制不满足条件的数据是否显示

上节课的多表查询就是一个内连接查询 （from 数据来源1,数据来源2）

为emp数据表增加一条数据

INSERT INTO emp(empno,ename,job,sal) VALUES(‘1001’,’老王’,’总裁’,6666.6);

查询出雇员的编号、姓名、职位、薪资、部门的位置和部门编号

SELECT empno,ename,job,sal,loc,d.deptno FROM emp e,dept d WHERE e.deptno=d.deptno;

发现了emp表中的张三的信息没有显示，dept表中的40号部门也没有显示，原因是以上的多表查询是一个内连接查询，内连接查询的特征是满足过滤条件的数据才会被显示

◆对于emp表中的张三的数据来说，deptno字段为null，很显然不满足 WHERE e.deptno=d.deptno，自然就不显示班长的数据。

◆对于dept表中的40号部门的数据来说，所有雇员中没有一个雇员是40号部门，所以40号部门也不显示了，以上是内连接查询的特征导致。

除了默认（FROM 数据表,数据表）方式我们还可以使用关键字INNER JOIN关键字实现内连接查询。

使用INNER JOIN实现内连接查询

SELECT empno,ename,job,sal,dname,loc,d.deptno FROM emp e INNER JOIN dept d ON e.deptno=d.deptno;

查询出雇员的编号、姓名、职位、部门名称、部门位置、薪资等级

使用新的方式实现三张数据表的内连接查询。

SELECT empno,ename,job,sal,dname,loc,grade FROM emp e INNER JOIN dept d ON e.deptno=d.deptno INNER JOIN salgrade s ON e.sal BETWEEN s.losal AND s.hisal;

第一种内连接查询：在FROM子句中查询多个数据来源，在WHERE子句中去掉笛卡尔积

第二种内连接查询：使用 INNER JOIN 连接数据来源，在ON关键字之后去掉笛卡尔

总结

  1. 内连接查询，只显示满足过滤条件的数据，我们之前的多表查询就是一个内连接查询。
  2. 你要熟悉使用第二种内连接查询。

外链接查询

内连接查询的特征是不满足条件的数据不显示，只有满足条件的数据才显示。

如果要求不满足条件的数据也显示出来，需要使用到外连接查询，外连接查询又分为：

·左外连接查询：可以让左表不满足条件的数据也显示

·右外连接查询：可以让右表不满足条件的数据也显示

·全外连接查询：可以让左表和右表不满足条件的数据都行显示

实现左外链接查询

SELECT empno,ename,job,sal,d.deptno,dname,loc FROM emp e LEFT OUTER JOIN dept d ON e.deptno=d.deptno;

右外链接查询

SELECT empno,ename,job,sal,d.deptno,dname,loc FROM emp e RIGHT OUTER JOIN dept d ON e.deptno=d.deptno;

全外链接查询

SELECT empno,ename,job,sal,d.deptno,dname,loc FROM emp e FULL OUTER JOIN dept d ON e.deptno=d.deptno;
mysql不支持上面的全外连接，但是oracle支持。

mysql的全外连接

SELECT empno,ename,job,sal,d.deptno,dname,loc FROM emp e LEFT OUTER JOIN dept d ON e.deptno=d.deptno UNION SELECT empno,ename,job,sal,d.deptno,dname,loc FROM emp e RIGHT OUTER JOIN dept d ON e.deptno=d.deptno;

总结

内连接查询的方式有两种：

· 直接在FROM之后跟上多张数据表，在WHERE条件中去掉笛卡尔积

· 使用 INNER JOIN 进行连接，在ON 之后去掉笛卡尔积

外连接查询：

·左外连接：使用 LEFT OUTER JOIN 实现，在ON之后去掉笛卡尔积

·右外连接：使用 RIGHT OUTER JOIN 实现，在ON之后去掉笛卡尔积

·全外连接：使用 FULL OUTER JOIN 实现（在mysql中不支持，在oracle中支持），在ON之后去掉笛卡尔积

常用的函数

Mysql提供了很多函数，有几十个，但是常用的不多，函数就是数据库已经定义好的类似于我们Java中的一些方法。

◆COUNT(*|字段名)：统计出查询到的数据量

◆SUM(字段)：计算出指定字段的数据之和（字段是数字才有意义）

◆AVG(列名)：计算出指定字段的平均值（字段是数字才有意义）

◆MAX(字段)：统计出指定的字段中最大值（字段是日期或者数字才有意义）

◆MIN(字段)：统计出指定的字段中最小值（字段是日期或者数字才有意义）

◆NOW():取得当前日期

◆NVL(eExpression1, eExpression2) 参数 eExpression1, eExpression2
如果 eExpression1 的计算结果为 null 值，则 NVL() 返回 eExpression2。如果 eExpression1 的计算结果不是 null 值，则返回 eExpression1。eExpression1 和 eExpression2 可以是任意一种数据类型。如果 eExpression1 与 eExpression2 的结果皆为 null 值，则 NVL() 返回 .NULL.。

◆CONCAT(column1，column2，column3)连接显示

◆DATE_FORMAT(hiredate,’%Y-%m-%d’):日期格式化

请说明COUNT(*) 、COUNT(字段) 和COUNT(DISTINCT 字段)三者的区别？

·COUNT(*):统计所有的数据量，没有过滤功能

·COUNT(字段)：统计出指定字段不为null的数据量，有过滤功能

总结

其实上统计函数是为分组查询而服务的，之所以现在没有分组也能使用是因为把所有的数据当成了一组。具体什么是分组查询我们下节课分析。

分组统计查询

思考

1、 如何查询出每个职位的人数，比如说要查询salesman这个职位的人数？

2、 如何查询出每个职位的平均薪资 ？

考虑将雇员按照职位进行分组，之后使用我们学习过的统计函数对每个组进行统计。

其实之前学习的统计函数都是为分组查询服务的，什么是分组查询？如果要查询出每种职位的平均工资？如果如要查询出每个部门的人数？要实现类似的需求就要使用分组统计查询。基本的语法如下：

④ SELECT * 【显示数据】
① FROM 数据来源,数据来源… 【找到数据来源】
② WHERE 过滤条件 【过滤出满足条件的数据】
③ GROUP BY X(分组字段) 【GROUP BY 子句，对满足条件的数据进行分组】
⑤ ORDER BY 排序字段 排序方式 【指定是否按照排序的方式显示】

意思是将所有具有相同X字段值的记录（数据）放到一个分组里。

查询出每个职位的平均工资

SELECT job,AVG(sal) 平均薪资 FROM emp GROUP BY job;

统计出从事同一工作的雇员的平均薪资

查询出每个部门的名称及其人数

SELECT dname,COUNT(*) – 对分组后的数据进行统计 FROM emp e,dept d – 数据需要来自两张数据表 WHERE e.deptno=d.deptno – 去掉笛卡尔积 GROUP BY dname; – 按照部门分组

统计出每个职位的名称、人数、平均工资

SELECT job,COUNT(*) AS 人数,AVG(sal) 平均薪资 FROM emp GROUP BY job;

查询出每个部门的名称、人数、以及每个部门的雇员的最高和最低工资

SELECT dname,COUNT(*) 人数,MAX(sal) 最高薪资,MIN(sal) 最低薪资 FROM emp e,dept d WHERE e.deptno=d.deptno GROUP BY dname;

总结

  1. 统计函数是为分组查询服务的
  2. GROUP BY 字句实现了分组查询，对满足WHERE字句条件的数据按照指定的字段进行分组。

思考：

查询出 平均工资高于2000的职位 的 名称 和该职位的 平均薪资。

SELECT job,AVG(sal) avg

FROM emp

WHERE AVG(sal)>2000

GROUP BY job;

HAVING子句

思考：

查询出平均工资高于2000的职位的名称和该职位的平均薪资。

HAVING子句是对分组统计函数统计出的结果进行过滤的子句。

查询出平均工资高于2000的职位的名称和该职位的平均薪资。

·按照职位进行分组

·分组之后统计出每个职位的平均工资

·过滤出平均工资高于2000的职位

SELECT job,AVG(sal) FROM emp WHERE AVG(sal)>2000 GROUP BY job;

发现了在WHERE 子句中不能使用分组统计函数。此时我们的需求要对统计函数进行过滤（过滤出平均薪资大于2000的组），如果要对统计函数进行过滤需要使用到另外一个子句，该子句是HAVING子句。

⑤SELECT * – 查询数据
① FROM 数据来源 – 找到数据来源
② WHERE 条件判断 – 对找到的数据进行过滤
③ GROUP BY 分组字段 – 对满足条件的数据进行分组
④ HAVING 对统计函数进行判断 –对分组统计出的数据进行过滤
⑥ ORDER BY 排序字段 排序方式 – 控制数据的显示方式（升序或者降序）

使用HAVING子句实现

SELECT job,AVG(sal) FROM emp GROUP BY job HAVING AVG(sal)>2000;

查询出非销售人员的工作名称 以及 从事同一工作的雇员的月工资总和，并且要求从事同一工作的雇员的薪资总和大于5000，输出的结果按照雇员薪资之和升序显示。

①查询出非销售人员的职位 SELECT job FROM emp WHERE job<>‘salesman’;
②按照职位进行分组 SELECT job,SUM(sal) FROM emp WHERE job<>‘salesman’ GROUP BY job;
③ 使用having字句过滤出薪资综合大于5000的 SELECT job,SUM(sal) FROM emp WHERE job<>‘salesman’ GROUP BY job HAVING SUM(sal)>=5000;
④ 使用ORDER BY字句 实现按照薪资升序排序 SELECT job,SUM(sal) FROM emp WHERE job<>‘salesman’ GROUP BY job HAVING SUM(sal)>=5000 ORDER BY SUM(sal) ASC;

总结：请说明WHERE子句和HAVING子句的区别？

·WHERE子句在GROUP BY 子句之前执行，不能在WHERE 子句中使用统计函数

·HAVING子句在GROUP BY之后执行，可以在HAVING子句中对统计函数进行过滤。
