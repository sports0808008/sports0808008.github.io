1.反射

定义:

Java反射是指在Java程序运行时动态地获取、检测和使用类的信息以及操作对象的能力。通过Java反射，我们可以在运行时获取类的属性、方法、构造函数等信息，并且可以动态地创建对象、调用方法、访问和修改属性，而不需要在编译时提前知道这些信息。

Java的反射机制主要依赖于java.lang.reflect包中的类和接口，其中一些核心类如下：

  • Class类：代表一个Java类或接口。可以通过Class.forName()方法根据类名获取对应的Class对象，或者使用.class来获取一个类的Class对象。

  • Field类：代表类的字段（成员变量），通过Class.getField()方法或者Class.getDeclaredField()方法可以获取字段的信息，包括字段类型、名称、修饰符等。

  • Method类：代表类的方法，通过Class.getMethod()方法或者Class.getDeclaredMethod()方法可以获取方法的信息，包括方法名称、参数类型、返回类型、修饰符等。

  • Constructor类：代表类的构造函数，通过Class.getConstructor()方法或者Class.getDeclaredConstructor()方法可以获取构造函数的信息，包括构造函数参数类型、修饰符等。

Java反射机制使得我们可以在运行时通过类的反射信息进行动态操作，它提供了更大的灵活性和可扩展性。它可以用于实现动态代理、配置文件解析、依赖注入（DI）容器以及一些框架和工具的开发。

需要注意的是，反射虽然提供了很多便利，但由于涉及动态检查和调用，性能开销较大。因此，在性能敏感的场景下要谨慎使用，并避免滥用反射机制。

1.反射的概念

反射(Reflection): 反射是一种Java编程语言的功能。

JAVA反射机制是在运行状态中，对于任意一个类，都能够知道这个类的所有属性和方法；对于任意一个对象，都能够调用它的任意一个方法和属性；这种动态获取的信息以及动态调用对象的方法的功能称为java语言的反射机制。

反射就是把Java类中的各种成分(属性、普通方法、构造方法)映射成一个个的java对象。例如，一个类有：成员变量，方法，构造方法，包等等信息，利用反射技术可以对一个类进行解剖，把各个组成部分映射成一个个对象。

Java基本的组成单元由类组成的，类(属性、普通方法、构造方法)，要想解剖一个类,必须先要获取到该类的字节码文件对象。而解剖使用的就是Class类中的方法.所以先要获取到每一个字节码文件对应的Class类型的对象。

2.反射的作用

反射是Java中的高级特性，在各种Java框架中都需要使用反射。所以，就算你将来很长一段时间不使用反射，但你使用的框架都大量使用了反射，所以想深入学习框架，那么就一定要学习反射。

框架通常通过反射来识别一个对象的“类型信息”。当你传递给框架一个对象时，框架会通过反射来了解对象的真实类型（对象实体的类型，而不是引用的类型），这个类型有几个构造器，有什么样的属性，有什么样的方法。还可以通过反射调用构造器，调用方法，对属性进行读写操作。

  2. setProperty(String key, String value): 设置属性的键值对。
  3. getProperty(String key): 根据键获取属性值。
  4. getProperty(String key, String defaultValue): 根据键获取属性值，如果键不存在则返回默认值。
  5. remove(Object key): 根据键移除属性。
  6. containsKey(Object key): 检查是否包含指定的键。
  7. keySet(): 获取所有属性键的集合。
  8. values(): 获取所有属性值的集合。

  10. store(OutputStream outputStream, String comments): 将属性保存到输出流中，可添加注释。

这些方法可以帮助你创建、获取、修改、删除属性，以及获取属性键值对的集合。
